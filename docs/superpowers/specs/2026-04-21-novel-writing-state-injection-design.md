# 小说写作状态注入系统设计文档

> 解决批量创作时的上下文不连贯和AI味道问题，通过状态注入机制确保连贯性，通过风格指导消除AI痕迹。

---

## 问题背景

### 用户痛点

1. **上下文跳跃**：章节之间场景转换太突然，读者感到"断层"
2. **细节遗忘**：AI写到后面忘了前面设定（人物能力、场景细节）
3. **风格机械**：写作模式化，每章结构太固定，读起来像"模板生产"
4. **情感平淡**：人物的情感反应不够真实，对话像"机器人"

### 当前方式

- 用户使用批量创作方式（一次写5-10章）
- 依赖skill的reference文件让AI读取总结
- 但不确定AI是否真正使用了总结文件
- chapter-summary.md主要记录事件，缺少情感、氛围等软信息

---

## 设计方案

### 核心思路

**状态注入型写作skill**：在现有skill基础上新增"状态注入机制"

- 强制上下文读取：每次写作前，skill强制要求读取状态文件
- 状态文件分层：长期状态（事件）+ 短期状态（场景/情绪/悬念）+ 风格指导
- 强制验证：写作输出时附带状态验证声明
- 自动更新：写作后自动更新状态文件

## 三层状态文件设计

### 第一层：长期状态（chapter-summary.md）

**用途**：记录整体进度，跨多个创作批次使用

**内容**：
- 境界进度、功法、团队成员
- 世界观进度、已揭示伏笔
- 整体剧情走向、已完成章节摘要

**更新频率**：每章写完后检查，如有重大进展（境界突破、团队变化）则更新

**文件来源**：复用现有文件，内容结构保持不变

```markdown
## 一、对话口语化

### 问题
AI写的对话太正式、太完整，像写论文而不是人在说话。

### 解决原则

1. **加入语气词**：哼、呵、啧、唔、呃...
2. **使用不完整句**：人说话经常中途打断、省略
3. **加入停顿和犹豫**：用省略号表示思考
4. **方言/习惯用语**：符合人物性格的表达

### 示例对比

**AI写法（问题）**：
```
林衍说："这个遗迹可能存在危险，我们需要谨慎行事。"
楚瑶回答："我同意你的看法，我们可以先用阵法探查内部情况。"
```

**自然写法（改进）**：
```
林衍皱眉："这遗迹......有点不对劲。"
楚瑶轻点头："嗯。我先用阵法探一下？"
```

### 具体技巧

| 技巧 | 示例 |
|------|------|
| 语气词开头 | "呵，你倒是自信。" |
| 短句断句 | "等等——那边有人。" |
| 省略回答 | "嗯。"、"行。" |
| 犹豫停顿 | "这个......让我想想。" |
| 情绪渗入 | "（冷笑）呵。" |

---

## 二、描写节奏变化

### 问题
AI描写每个场景的密度差不多，没有重点和节奏变化。

### 解决原则

1. **重点场景详写**：战斗高潮、情感转折、重大发现
2. **过渡场景简写**：移动、日常、铺垫
3. **用"密度标记"控制**：在章节框架中标记描写密度

### 示例对比

**AI写法（问题）**：所有场景描写密度相同
```
林衍走进废墟。废墟中有很多碎石和残骸。他仔细观察四周。
他发现一个灵气池。灵气池散发着微弱的光芒。
```

**自然写法（改进）**：重点详写，过渡简写
```
林衍踏入废墟。（简写过渡）

他的目光骤然凝固——

前方，一道幽蓝的光芒从地面升腾。那是一个灵气池。

池面泛着涟漪，仿佛某种沉睡的力量正在苏醒。林衍屏住呼吸，他能感到灵气在召唤他......

（详写发现场景）
```

### 描写密度规则

| 场景类型 | 描写密度 | 字数占比 |
|----------|----------|----------|
| 战斗高潮 | 高密度 | 每分钟动作用50-100字 |
| 情感转折 | 高密度 | 情绪变化用100-200字 |
| 重大发现 | 高密度 | 发现过程用100-150字 |
| 场景过渡 | 低密度 | 一句话带过 |
| 日常铺垫 | 低密度 | 2-3句简述 |

---

## 三、心理真实感

### 问题
AI写心理活动太分析化，像"我现在很愤怒，因为..."，不像真实人的思考。

### 解决原则

1. **用感受代替分析**：写"胸口发紧"、"手心出汗"，不写"我感到紧张"
2. **用碎片代替完整**：心理活动是片段的，不是完整论述
3. **用行动暗示心理**：用动作、表情暗示心理状态

### 示例对比

**AI写法（问题）**：
```
林衍感到非常愤怒。他想起了赵虎被王霸欺负的场景，心中充满了复仇的欲望。他决定要让王霸付出代价。
```

**自然写法（改进）**：
```
林衍的拳头攥紧，指节发出轻响。

赵虎......被压在地上的画面，在他脑海中闪过。

"王霸。"他咬牙，一字一顿。

"十倍奉还。"
```

### 心理描写技巧

| 技巧 | 示例 |
|------|------|
| 身体感受 | 胸口发紧、喉咙干涩、手心出汗 |
| 碎片画面 | 画面闪过、声音回响、模糊记忆 |
| 行动暗示 | 攥拳、咬牙、目光锐利 |
| 简短判断 | "不行。"、"有机会。" |

---

## 四、转折自然化

### 问题
AI写的转折太平滑、太"顺利"，缺少真实世界的突兀感。

### 解决原则

1. **情绪突变**：情绪变化往往是突然的，不需要完整铺垫
2. **场景跳跃**：真实世界的时间流逝是跳跃的，不必每个过渡都写
3. **意外打断**：用突发事件打断当前进程

### 示例对比

**AI写法（问题）**：情绪变化太平滑
```
林衍原本很平静，但听到王霸的话后，他逐渐感到愤怒，最终决定反击。
```

**自然写法（改进）**：情绪突变
```
林衍听完，脸色没有变化。

然后——

他的眼神骤然锐利，如刀锋出鞘。

"你说什么？"
```

### 转折技巧

| 类型 | 技巧 | 示例 |
|------|------|------|
| 情绪突变 | 用"骤然"、"突然" | 他的笑容骤然消失 |
| 场景跳跃 | 直接切到新场景，无需过渡 | 三天后。遗迹入口。 |
| 意外打断 | 用突发事件打断 | "等等——有人来了。" |

## 五、综合应用示例

### 场景：主角发现敌人埋伏

**AI写法（问题）**：
```
林衍走进山谷。他仔细观察四周的环境，发现有些不对劲。经过分析，他判断可能有敌人埋伏。他感到紧张，对楚瑶说："我们要小心，可能有埋伏。"楚瑶表示同意。
```

**自然写法（改进）**：
```
林衍踏入山谷。

脚步声忽然停住。

他的目光扫过两侧山壁——太安静了。连风声都没有。

"楚瑶。"他低声。

"嗯？"

"......不对劲。"

远处，一道阴影动了。
```

## 应用规则

创作时，必须对照本指导检查：

1. 对话是否有口语化元素？
2. 描写密度是否符合场景重要性？
3. 心理活动是用感受而非分析？
4. 转折是否有突兀感？
```

## 强制读取与验证机制

### SKILL.md流程更新

```markdown
## 写作流程（硬编码执行）

当用户请求创作时，按以下流程执行：

### 第一阶段：状态注入（必须执行）

1. **读取状态文件**
   - 读取 `references/writing-snapshot.md`（获取当前场景、情绪、悬念）
   - 读取 `references/writing-style-guide.md`（获取风格去AI指导）
   - 读取 `references/chapter-summary.md`（获取长期上下文）

2. **状态确认**
   - 在内部确认关键状态信息已读取
   - 如状态文件不存在或过旧，提醒用户需要先创建/更新

### 第二阶段：构建框架

3. **查阅其他必要资料**
   - 根据本章内容查阅：world-settings、characters、foreshadowing-tracker
   - 检查是否有伏笔可以铺垫或揭示

4. **构建章节框架**
   - 确定本章爽点类型
   - 确定结尾悬念
   - 标记描写密度（高密度场景vs低密度场景）
   - 确定是否有伏笔需要处理

### 第三阶段：正式创作

5. **创作章节正文**
   - 按章节结构规范展开（2000-2500字）
   - **应用风格去AI指导**（对话口语化、描写节奏、心理真实感、转折自然化）
   - 融合辰东+西红柿风格要素

6. **字数验证**（必须执行）
   - 统计正文字数（不含标题和【本章要点】）
   - 如不达标，扩充/精简（最多2轮）

### 第四阶段：输出与更新

7. **输出章节**
   - 输出正文内容
   - **附带状态验证声明**
   - 附带风格验证声明
   - 附带【本章要点】总结

8. **更新状态文件**（必须执行）
   - 更新 `references/writing-snapshot.md`（新的结束状态）
   - 如有境界突破/团队变化，更新 `references/chapter-summary.md`
   - 如有新伏笔，更新 `references/foreshadowing-tracker.md`
```

## 自动更新机制

### writing-snapshot.md更新规则

每章写完后，skill强制执行以下更新：

1. **场景状态**：更新为新章节结束时的场景
2. **主角状态**：更新情绪、体力、灵气储备
3. **悬念状态**：记录新章节结尾悬念
4. **时间线锚点**：更新距离上一章的时间

### chapter-summary.md更新规则

只在以下情况更新：

- 境界突破
- 团队成员变化（加入/离开）
- 重大剧情节点完成
- 伏笔揭示

### foreshadowing-tracker.md更新规则

只在以下情况更新：

- 埋下新伏笔
- 揭示已有伏笔

---

## 用户无需额外操作

用户只需说"写一章"或"写第X章"，skill自动执行：

- 读取状态文件（无需用户触发）
- 应用风格指导（自动应用）
- 输出验证声明（透明显示，无需用户确认）
- 更新状态文件（自动更新）

用户可以查看验证声明确认AI确实读取了状态，但无需主动操作。

## 预期效果

### 连贯性改善

- 状态文件强制读取 → AI不会遗忘上下文
- 短期状态记录软信息 → 情绪、氛围连续
- 时间线锚点 → 时间跳跃不会混乱

### AI痕迹消除

- 对话口语化指导 → 人物说话更自然
- 描写节奏控制 → 不再均匀机械
- 心理真实感 → 用感受代替分析
- 转折自然化 → 有突兀感，不平滑

### 上下文控制

- 状态文件简洁 → 不会导致上下文溢出
- 分层结构 → 长期/短期分离，减少每次读取量
- 自动更新 → 无需用户手动维护
