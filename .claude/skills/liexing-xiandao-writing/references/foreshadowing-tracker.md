## 伏笔记录格式

每条伏笔记录包含以下字段：

| 字段 | 说明 |
|------|------|
| **编号** | 伏笔唯一编号，格式：VP-XXX |
| **类型** | 身世/世界/势力/物品/伙伴/其他 |
| **埋下章节** | 伏笔首次出现的章节 |
| **伏笔内容** | 具体埋下的线索或暗示 |
| **预期揭示** | 计划在哪一卷/章节揭示 |
| **揭示章节** | 实际揭示的章节（填写后表示已闭环） |
| **状态** | 已埋下/已揭示/已闭环 |

```
**揭示内容**：[具体揭示的内容]
**揭示方式**：[通过什么方式揭示：对话/回忆/场景/战斗等]
```

## 使用说明

1. **创作新章节时**：检查本文件，避免遗漏已有伏笔
2. **埋下新伏笔时**：立即添加新记录
3. **揭示伏笔时**：更新状态并添加揭示详情
4. **每卷结束时**：统计本卷埋下/揭示的伏笔数量
5. **小说完成前**：检查所有伏笔是否都已闭环

---

## 注意事项

- 同一伏笔可以在多个章节铺垫，但只在一条记录中追踪
- 伏笔的预期揭示位置可以根据剧情调整，但需要同步更新本文件
- 避免伏笔过多导致后期无法全部揭示
- 避免伏笔揭示时与埋下时的设定矛盾
